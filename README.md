# k8s-altinity-click

## Overview

This repository shows how to **deploy a ClickHouse cluster** on a local Kubernetes cluster using **Kind** and the **Altinity ClickHouse Operator**. It also includes a **VictoriaMetrics stack** for monitoring ClickHouse metrics. The focus is on the Kubernetes resources and operator configuration; the example Go application is provided only as a minimal client.

## Prerequisites

| Tool | Minimum Version |
|------|-----------------|
| Docker | 20.10 |
| Kind | 0.20 |
| kubectl | 1.27 |
| Go | 1.22 |
| curl | any |

Make sure the above tools are installed and available in your `PATH`.

## Quick Start

```bash
# Clone the repository
git clone https://github.com/your-org/k8s-altinity-click.git
cd k8s-altinity-click

# Create a Kind cluster
./cluster-setup.sh create

# Install the Altinity ClickHouse Operator and the ClickHouse cluster
./setup-click.sh install

# (Optional) Deploy VictoriaMetrics for monitoring ClickHouse
kubectl apply -f https://raw.githubusercontent.com/VictoriaMetrics/helm-charts/master/charts/victoria-metrics-single/manifests/victoria-metrics-single.yaml

# Verify the deployment
kubectl get pods -n default
```

The repository also contains a minimal Go client (`app/`) that can be used to verify connectivity, but the primary purpose of this project is the Kubernetes and monitoring setup.

## Detailed Steps

### 1. Create a Kind Cluster

The `cluster-setup.sh` script wraps Kind commands:

```bash
./cluster-setup.sh create   # creates a cluster named "altinity-click"
./cluster-setup.sh delete   # removes the cluster
```

It also configures the local Docker registry so that the built Go application image can be loaded into Kind without pushing to an external registry.

### 2. Install Altinity ClickHouse Operator

`setup-click.sh` performs the following actions:

1. Installs the Altinity ClickHouse Operator via Helm.
2. Applies the `ClickHouseInstallation` CRD.
3. Creates a `ClickHouseInstallation` resource that defines a ClickHouse cluster with 2 shards and 2 replicas per shard.

You can customize the ClickHouse configuration by editing the generated `clickhouse.yaml` file before applying it.

### Deploy ClickHouse with Altinity Operator

`setup-click.sh` installs the Altinity ClickHouse Operator via Helm and applies the `ClickHouseInstallation` CRD. It then creates a `ClickHouseInstallation` resource (`k8s/clickhouse.yaml`) that defines a cluster with 2 shards and 2 replicas per shard. Customize the ClickHouse configuration by editing this manifest before applying it.

```bash
./setup-click.sh install
```

### (Optional) Deploy VictoriaMetrics for Monitoring

VictoriaMetrics provides a lightweight, highвЂ‘performance monitoring solution. Install a singleвЂ‘node instance:

```bash
kubectl apply -f https://raw.githubusercontent.com/VictoriaMetrics/helm-charts/master/charts/victoria-metrics-single/manifests/victoria-metrics-single.yaml
```

Add a `ServiceMonitor` or configure the ClickHouse exporter to scrape metrics.

## Cleaning Up

To remove everything:

```bash
kubectl delete -f k8s/   # removes app resources
./setup-click.sh uninstall   # removes operator and ClickHouse CRs
./cluster-setup.sh delete   # deletes Kind cluster
```
