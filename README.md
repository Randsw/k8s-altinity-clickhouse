# k8s-altinity-click

## Overview

This repository demonstrates how to **deploy a ClickHouse cluster** on a local Kubernetes cluster using **Kind** and the **Altinity ClickHouse Operator**. It also provides an **example Go application** that connects to the ClickHouse cluster, as well as a **VictoriaMetrics stack** for monitoring ClickHouse metrics.

The setup is fully automated with shell scripts and Kubernetes manifests, making it easy to spin up the whole environment locally for development or testing purposes.

## Prerequisites

| Tool | Minimum Version |
|------|-----------------|
| Docker | 20.10 |
| Kind | 0.20 |
| kubectl | 1.27 |
| Go | 1.22 |
| curl | any |

Make sure the above tools are installed and available in your `PATH`.

## Quick Start

```bash
# Clone the repository
git clone https://github.com/your-org/k8s-altinity-click.git
cd k8s-altinity-click

# Create a Kind cluster
./cluster-setup.sh create

# Deploy Altinity ClickHouse Operator and ClickHouse cluster
./setup-click.sh install

# Deploy the example Go application
kubectl apply -f k8s/deployment.yaml -f k8s/service.yaml -f k8s/ingress.yaml

# Deploy VictoriaMetrics stack (optional, for monitoring)
kubectl apply -f https://raw.githubusercontent.com/VictoriaMetrics/helm-charts/master/charts/victoria-metrics-single/manifests/victoria-metrics-single.yaml

# Verify everything is running
kubectl get pods -n default
```

The Go application will be reachable at `http://localhost` (or via the Ingress host you configure). It will automatically create a simple table in ClickHouse and insert a few rows.

## Detailed Steps

### 1. Create a Kind Cluster

The `cluster-setup.sh` script wraps Kind commands:

```bash
./cluster-setup.sh create   # creates a cluster named "altinity-click"
./cluster-setup.sh delete   # removes the cluster
```

It also configures the local Docker registry so that the built Go application image can be loaded into Kind without pushing to an external registry.

### 2. Install Altinity ClickHouse Operator

`setup-click.sh` performs the following actions:

1. Installs the Altinity ClickHouse Operator via Helm.
2. Applies the `ClickHouseInstallation` CRD.
3. Creates a `ClickHouseInstallation` resource that defines a ClickHouse cluster with 2 shards and 2 replicas per shard.

You can customize the ClickHouse configuration by editing the generated `clickhouse.yaml` file before applying it.

### 3. Deploy the Example Application

The Go application connects to ClickHouse using the DSN defined in `app/config.go`. By default it uses the service name `clickhouse` which is created by the operator.

Build and push the Docker image:

```bash
docker build -t localhost:5000/altinity-click:latest ./app
kind load docker-image localhost:5000/altinity-click:latest --name altinity-click
```

Then apply the Kubernetes manifests:

```bash
kubectl apply -f k8s/deployment.yaml -f k8s/service.yaml -f k8s/ingress.yaml
```

### 4. (Optional) Deploy VictoriaMetrics Stack

VictoriaMetrics provides a lightweight, highвЂ‘performance monitoring solution. The following command installs a singleвЂ‘node VictoriaMetrics instance:

```bash
kubectl apply -f https://raw.githubusercontent.com/VictoriaMetrics/helm-charts/master/charts/victoria-metrics-single/manifests/victoria-metrics-single.yaml
```

You can then scrape ClickHouse metrics by adding a `ServiceMonitor` or configuring the ClickHouse exporter.

## Cleaning Up

To remove everything:

```bash
kubectl delete -f k8s/   # removes app resources
./setup-click.sh uninstall   # removes operator and ClickHouse CRs
./cluster-setup.sh delete   # deletes Kind cluster
```
